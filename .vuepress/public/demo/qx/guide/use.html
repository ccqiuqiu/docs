# 基本配置

项目初始化完成后，在 /src/main.js 文件已经导入了qianxun依赖库，主要代码片段如下


 

 
































// 使用qianxun库
import qianxun from 'qianxun'

Vue.use(qianxun, {
  router: {
    routes, // 子路由
  },
  store: {
    modules, // vuex子模块
  },
  components: {
    title: Logo, // 网站标题
    // nav: Logo, // 网站导航部分扩展
    // login: Logo, // 登录页面
    // home: Logo, // 首页
  },
  actions: {
    login: 'system/login', // 登录的action
    // captcha: '', // 获取验证码的action
    // logout: '', // 退出的action
    userInfo: 'system/getUserInfo' // 获取用户信息
  },
  http: {
    statusName: 'success',
    successCode: true,
  },
  // 主框架初始化数据的时候执行，如果返回promise，会加入到框架初始化的promise同步执行(支持数组)
  beforeCreated: () => {
    console.log('beforeCreated')
  },
  // 主框架数据初始化后执行
  afterCreated: () => {
    console.log('afterCreated')
  },
})

# 完整配置

通过 Vue.use(qianxun, config) 来初始化qianxun，config可以指定使用时候的参数配置。

注意

  • App的挂载以及路由和vuex的注册由qianxun框架完成，只需要在配置的时候传入路由列表和子模块即可
  • 千寻的整体布局是固定的，默认传入的routes会出现在页面固定的内容区域

# ui.size

  • Type: String
  • Default: - 全局定义组件的尺寸，可选： medium / small / mini

# ui.forceMultiTab

  • Type: Boolean
  • Default: false 强制仅参数不同的路由在新tab页面打开。默认情况下当打开了一个url为user?id=1的tab页，再打开user?id=2的时候会替换掉前面那个tab页， 设置forceMultiTab为true后，会在新tab页打开user?id=2。

# router.mode

  • Type: String
  • Default: 'history' 路由的模式，同vue-router的配置

# router.routes

  • Type: Array
  • Default: - 路由配置数组，路由配置数组，这个数组将作为整个框架的子路由。路由对象的属性如下：
{
  path: String, // 路由的访问路径
  component: Component, // 路由对应的组件
  meta: { // 路由元信息配置
    title: String, // 路由对应的页面标题
  },
}

注意

  • 路由的代码组织参见项目router目录
  • 建议分模块拆分路由，路由的path命名方式建议 /模块名/路由
  • 组件建议使用异步组件的方式配置 () => import(/* webpackChunkName: "模块名" */ '../views/demo/XtTableDemo.vue')
  • webpackChunkName相同的组件会被打包成一个js文件，所以一般情况，一个模块下的组件用同样的webpackChunkName以合并打包到一个文件, 或者多个小模块用同一个名字合并成一个文件，总之一切按照最终输出的js文件大小来划分，在js文件的数据和大小之前平衡选择。

# store.modules

  • Type: Object
  • Default: - 对应vuex的模块配置。具体使用参照vuex模块 (opens new window)

注意

  • 子模块的代码组织参见项目的store目录
  • 子模块的 namespaced 一定要设为true，防止命名冲突

# store.cache

  • Type: Object
  • Default: - 缓存store里面的值。默认情况下，页面刷新后会丢失store的状态，如果有属性需要缓存，请设置这个对象。格式为：
{
 menuTabs: state.menuTabs,
 ... // 更多
}

# components.title

  • Type: Component
  • Default: XtLogo 网站Logo部分，可以通过配置这个组件，来替换框架默认的标题位置

# components.userIcon

  • Type: Component
  • Default: - 网站头部，用户图标组件，默认情况下，只显示用户的图标，如果需要在图标上增加以下操作，如显示一个下拉菜单，可以定义这个组件。这个组件将以插槽的方式替换掉 原本的用户图标

# components.nav

  • Type: Component
  • Default: - 网站头部导航位置，会显示在退出按钮的左边，可以定义更多的操作按钮

# components.login

  • Type: Component
  • Default: XtLogin 登陆页面，qianxun默认有一个登陆页面，如果想要重写登陆页面，可以通过这个参数指定登陆页面的组件

# components.home

  • Type: Component
  • Default: XtHome 登陆页面，qianxun默认有一个首页，如果想要重写首页，可以通过这个参数指定登陆页面的组件

# actions.login

  • Type: String
  • Default: login 默认登陆页面，点击登录按钮请求的action名称。如果使用默认登陆页面，需要设置这个值，以请求正确的接口

# actions.captcha

  • Type: String
  • Default: captcha 默认登陆页面，获取验证码请求的action名称。如果使用默认登陆页面，需要设置这个值，以请求正确的接口

# actions.logout

  • Type: String
  • Default: logout 退出登录的时候调用的action。
参数名 类型 描述 默认值
actions Object 替换框架默认的网络请求action -
actions.captcha String 获取验证码的action -
actions.logout String 退出的action logout
actions.userInfo String 获取用户信息的action getUserInfo
http Object 网络请求相关配置 -
http.statusName String 判断接口的状态字段 code
http.successCode Any 接口返回成功的状态值 200
http.messageName String 异常消息字段名 message
http.noAuthCode Number 没有权限的异常代码 401
http.needLoginCode Number 需要登录的异常代码 403
beforeCreated Function 主框架初始化数据的时候执行的函数 -
afterCreated Function 主框架初始化数据完成后执行的函数 -
